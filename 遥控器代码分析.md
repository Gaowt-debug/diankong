**遥控器部分**
**整体设计思路**

本项目基于 STM32F103 平台，实现了一个多按键交互 + 多路角度采集 + 串口通信的实时控制系统。
系统以 1ms 定时器中断为调度核心，完成按键消抖和周期性传感器读取；在主循环中处理按键动作和角度数据的组帧发送。
整体采用状态机 + 前后台架构，利用全局标志位和缓冲区实现模块解耦，同时通过 I²C 多路复用器（TCA9548）扩展出 4 路 AS5600 角度传感器，实现了关节角度的连续监测与变化上报。

**代码框架**
1. 硬件抽象层
stm32f10x.h 标准外设库

各外设驱动模块：Delay、OLED、hc-sr04、usart、pca9685、other、tca9548

AS5600 读取函数（通过 I²C 多路复用器切换通道）

2. 模块初始化 (all_init())
配置 GPIO 按键输入（PA8/11/12/15, PB3/4/7），禁用 JTAG 保留 SWD

初始化 2 路串口（USART1 用于上位机通信，USART2 用于数据转发）

初始化定时器 TIM3（1ms 中断）作为系统滴答

初始化 I²C 多路复用器及 AS5600

3. 后台中断服务
TIM3_IRQHandler

1ms 定时：为 7 个按键的消抖计数器（button_sta）递增/递减

每 150ms 周期：当 lock == 1 时，读取 4 路 AS5600 角度并做标定换算，更新全局 agl[] 数组

USART1_IRQHandler：接收上位机指令（预留）

USART2_IRQHandler：接收外部数据并直接转发到 USART1

4. 前台主循环
按键动作处理
遍历 button_sta[] 消抖状态，识别下降沿（按下）和上升沿（松开），并通过 USART_sendbit() 发送对应命令字符（M/N/i/j 等），同时控制全局锁 lock。

角度变化上报
当 agl[] 与历史 agl_la[] 不一致时，按帧格式 [ angle0,angle1,angle2,angle3 ] 通过 USART1 发送，并更新历史值。

**重点设计部分**
1. 按键消抖方案设计，定时器计数和主函数状态变化响应配合做到既稳定又减少资源占用
2. 角度变化上报也采用有变化在发送数据，减少机器人和遥控器的占用
3. 磁编码器与舵机占空比的转换和转动范围内遇到由极大到极小突变的处理
4. 三蓝牙转发实现不进行大型修改的情况下将机械臂遥控独立出来

**方案启发来源**

距离比赛开始约一周时实在受不了用按键操作机械臂造成的笨重，受到曾经看过的视频启发想出制作简易的操作杆，当时时间比较紧，当晚制定出磁编码器和蓝牙转发方案，并让一位队友进行遥控器制作，同时我继续进行视觉追踪部分调试（最终失败），快递到后大概只剩下五天时间，我尽可能不对车体代码进行大改，在原本手机蓝牙串口遥控基础上在传输中途加入遥控器转发合并，整个方案完全启用是在第一天比赛下午，为我们的拾取提供了极大提升。

**难点问题和解决方案**
1. i2c多路复用问题，因为磁编码器地址相同，不得不使用复用器并鞭策ai调试代码
2. 手机和机械臂遥控器数据合并，采用车与遥控器连接，遥控器与手机连接的方案，当然缺点是扩大了蓝牙不稳定的问题，也是最终被淘汰的原因
3. 角度传感器选择，淘宝搜索角度传感器，在一众方案中最后选择了灵敏简便的磁编码器
4. 电路构建，由于时间紧张，电路构建使用了洞洞板焊接，降压使用主控板载3.3稳压
5. 蓝牙主从连接，受到另一队的帮助得到主从一对蓝牙模块并帮助我们解决了该问题，后续自主解决了波特率问题
6. 基于操作需求设计了锁定按钮和上一版控制方案的兼容
7. 其他小难点：编码器和舵机之间角度换算，按键消抖，通信代码的调试
